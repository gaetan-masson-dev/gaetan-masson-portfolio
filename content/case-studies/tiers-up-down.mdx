## Overview

A SaaS platform had three pricing tiers (Free, Pro, Enterprise) but struggled with conversions from free to paid. Free users didn't understand the value of upgrading, and the upgrade experience was confusing. We needed to improve feature discovery and create clear paths to paid tiers.

## The Challenge

Despite strong product-market fit, conversion metrics were concerning:

- Only 4% of free users upgraded to paid plans
- Users hitting plan limits often churned rather than upgraded
- Feature discovery was poor (users didn't know premium features existed)
- No clear differentiation between Pro and Enterprise tiers
- Upgrade flow was a simple "Choose a plan" page with no context
- Downgrade from Enterprise required contacting sales (created friction)

The business needed better conversion without compromising the free tier's value or creating a pushy experience.

## Research

We conducted research to understand the upgrade decision process:

### User Interviews

Interviewed 25 users across all tiers:

- **Free users**: Didn't know what premium features did or why they'd need them
- **Pro users**: Discovered features by accident, wished they'd upgraded sooner
- **Enterprise users**: Wanted flexibility to downgrade during seasonal lulls
- **Churned users**: Hit limits but didn't see value in upgrading

### Competitive Analysis

Studied how successful SaaS products handle tier differentiation and upgrades:

- Slack's gentle feature discovery approach
- Figma's contextual upgrade prompts
- Notion's progressive limit enforcement
- Dropbox's visual storage representations

### Analytics Deep-Dive

Analyzed 6 months of product usage data:

- Free users heavily used features that had premium versions they didn't know about
- 70% of users hit limits without attempting to upgrade
- Users who tried one premium feature were 5x more likely to convert
- Mobile users rarely encountered upgrade prompts

## Strategy

Developed a multi-faceted approach:

### Feature Discovery

Help users understand premium features before they hit limits:

- Contextual feature previews showing what's available in paid tiers
- "Used by Pro users" social proof for premium features
- Trial periods for individual features before requiring upgrade
- Educational tooltips explaining premium capabilities

### Soft Limits

Replace hard blockers with gentle nudges:

- Preview premium results but blur them with upgrade CTA
- Allow occasional premium actions on free plan (create appetite)
- Clear progress indicators showing plan usage
- Advance warning before hitting limits

### Value-Based Upgrades

Show value, not just features:

- ROI calculators for specific use cases
- Success stories from similar users who upgraded
- Comparison showing what's possible at each tier
- Time/efficiency savings from premium features

## Solution

### In-App Feature Discovery

Redesigned how users discover premium features:

<Figure
  src="/images/tiers-up-down/feature-discovery.png"
  alt="Feature discovery interface"
  caption="Contextual previews showing premium features where users work"
/>

#### Premium Feature Previews

- Inline cards showing "This premium feature can..." with benefits
- "Try it once" options letting free users sample premium features
- Usage examples from similar users' workflows
- Clear tier indicators without being pushy

#### Feature Tours

- Onboarding flow highlighting features relevant to user's use case
- Role-based tours (marketer sees marketing features, designer sees design features)
- Dismissible but easily accessible later

### Contextual Upgrade Prompts

Created situation-specific upgrade moments:

<Figure
  src="/images/tiers-up-down/upgrade-prompt.png"
  alt="Contextual upgrade prompt"
  caption="Upgrade prompts tied to specific user goals and contexts"
/>

#### Smart Timing

- Triggered when user attempts premium action (not arbitrary)
- Shown after user completes successful task (positive moment)
- Limited frequency to avoid annoyance
- Personalized to user's actual usage patterns

#### Clear Value Proposition

- Specific to current task: "Upgrade to Pro to export in multiple formats"
- Shows exactly what they'll unlock
- Comparison with current plan limitations
- Simple, single-CTA design

### Improved Upgrade Flow

Redesigned the upgrade experience:

<Figure
  src="/images/tiers-up-down/upgrade-flow.png"
  alt="Upgrade flow interface"
  caption="Streamlined upgrade flow with clear plan comparison"
/>

#### Plan Comparison

- Side-by-side comparison with current plan highlighted
- Feature explanations, not just lists
- Usage-based recommendations ("Based on your activity, Pro fits best")
- FAQ addressing common concerns

#### Flexible Checkout

- Monthly/annual toggle with savings calculation
- Team size selection for Pro tier
- Add-ons (storage, seats) clearly presented
- Trial period option for annual plans

### Downgrade Experience

Made it easy to adjust plans:

- Self-service downgrade (no sales calls required)
- Clear explanation of what they'll lose
- Option to pause instead of downgrade
- Save settings/data for 30 days if they want to return

## Testing & Iteration

Ran A/B tests on key elements:

### Feature Discovery

- Tested different preview styles (blurred vs. locked vs. demo mode)
- Measured impact on upgrade consideration
- Result: Demo mode performed best (45% engaged with preview)

### Upgrade Prompt Timing

- Tested showing prompts at different points in workflow
- Measured annoyance vs. conversion
- Result: Post-success moments converted 3x better than mid-task

### Plan Comparison Layout

- Tested vertical vs. horizontal comparison
- Measured comprehension and decision time
- Result: Vertical layout (cards) performed better on mobile

## Results

The new upgrade experience significantly improved conversions:

- Free-to-paid conversion increased from 4% to 11%
- Time from signup to first upgrade reduced from 90 days to 45 days
- Feature discovery improved (70% of users now try premium features on free plan)
- Upgrade cart abandonment decreased from 55% to 22%
- Customer lifetime value increased by 34%
- Enterprise downgrades became 60% less common (better tier fit upfront)

## Key Learnings

- **Show, don't tell**: Letting users preview premium features was far more effective than describing them
- **Timing is everything**: Upgrade prompts during moments of success converted much better than interruptions during work
- **Remove friction everywhere**: Self-service downgrade actually reduced downgrades by helping users choose better tiers initially
- **Personalization matters**: Usage-based recommendations significantly improved conversion by matching users to appropriate tiers
- **Gentle beats aggressive**: Soft limits and previews outperformed hard blockers and pushy CTAs
- **Mobile is different**: Upgrade flows needed significant simplification for mobile (where 40% of conversions happened)
